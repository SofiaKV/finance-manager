## 1. Unit-тестування

Було повністю покрито модулі бізнес-логіки та контролери.

### Протестовані модулі:
- BudgetsController  
- BudgetsService  
- AppController  
- AppService  

### Основні сценарії:
- Перевірка наявності та валідності Authorization header  
- Коректна передача userId у сервіси  
- Обробка помилок (401 Unauthorized, 404 Not Found)  
- Створення, оновлення та видалення бюджетів  
- Розрахунок значення spent  
- Фільтрація транзакцій за категорією, датами та типом  
- Коректність DTO при оновленні (ігнорування undefined)  
- Граничні значення дат startDate / endDate  

---

## 2. Інтеграційні тести

Інтеграційні тести перевіряють роботу сервісів разом з DAO-класами.

### Перевірено:
- Робота BudgetsService у зв’язці з BudgetDao та TransactionDao  
- Обчислення spend на основі реальних даних DAO  
- Оновлення бюджетів та перевірка переданих полів  
- Правильна поведінка при некоректних userId  
- Коректні типи значень (Date, number, string)
  
---

## 3. End-To-End тестування

E2E тести виконані через NestFastifyApplication + Supertest.

### Перевірені сценарії:
1. Реєстрація нового користувача  
2. Авторизація користувача  
3. Отримання токена  
4. Створення нового бюджету  
5. Отримання списку бюджетів  
6. Перевірка роботи реального HTTP маршруту → контролера → сервісу → DAO
